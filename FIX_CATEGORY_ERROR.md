# Исправление ошибки с полем category

## Проблема

При сохранении документов возникает ошибка:
```
null value in column "category" of relation "documents" violates not-null constraint
```

## Причина

В базе данных все еще существует старое поле `category` с ограничением NOT NULL, но код приложения теперь использует только `category_id`.

## Решение

Выполните SQL скрипт `fix_category_column.sql` в Supabase SQL Editor:

1. Откройте Supabase Dashboard
2. Перейдите в SQL Editor
3. Выполните скрипт `fix_category_column.sql`

Этот скрипт:
- Удалит старое поле `category` из таблицы `documents`
- Оставит только новое поле `category_id`

## Альтернативное решение

Если по какой-то причине нужно сохранить оба поля временно, можно изменить код, чтобы отправлять оба поля при создании документов. Но рекомендуется удалить старое поле `category`, так как оно больше не используется.
