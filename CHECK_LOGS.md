# Проверка логов ошибки 500

## Что произошло

Функция задеплоилась, но падает с ошибкой 500 (INTERNAL_SERVER_ERROR).

## Как проверить логи

### Способ 1: Через Vercel Dashboard

1. Откройте: https://vercel.com/dashboard
2. Найдите проект "artecobase"
3. Перейдите в **Deployments**
4. Откройте последний деплой (должен быть зеленым или с ошибкой)
5. Перейдите в раздел **Functions**
6. Найдите `api/sync-availability`
7. Откройте логи (кнопка "View Logs" или "Functions" → "View")

### Способ 2: Через ссылку на странице ошибки

На странице ошибки есть ссылка "check the logs" - перейдите по ней.

### Способ 3: Через URL

Можно попробовать:
```
https://artecobase.vercel.app/_logs
```

## Что искать в логах

Обычные ошибки:
1. **Missing environment variables** - отсутствуют переменные окружения
2. **Cannot find module** - проблемы с импортами
3. **Puppeteer error** - ошибки с браузером
4. **Timeout** - превышен лимит времени
