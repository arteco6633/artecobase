# Исправление ошибки 404 для API route

## Проблема

Получаете ошибку 404 при вызове `/api/sync-availability`

## Возможные причины

1. **Неправильный URL** - "your-project" это placeholder, нужен реальный URL
2. **Vercel не распознал API route** - возможно нужна другая структура
3. **Функция не задеплоилась** - проверьте логи деплоя

## Решения

### 1. Проверить реальный URL проекта

В Vercel Dashboard:
1. Откройте ваш проект
2. Скопируйте реальный URL (например: `artecobase.vercel.app`)
3. Используйте: `https://artecobase.vercel.app/api/sync-availability`

### 2. Проверить структуру API route

Для Vite проектов API routes должны быть в папке `api/` в корне проекта.

Текущая структура:
```
api/
  sync-availability.ts
```

Это правильно для Vercel.

### 3. Проверить логи деплоя

В Vercel Dashboard:
1. Перейдите в Deployments
2. Откройте последний деплой
3. Проверьте логи на ошибки

### 4. Альтернатива: Использовать другой формат

Если API route не работает, можно:
- Создать через Vercel Dashboard (Functions)
- Или использовать другой подход

## Быстрая проверка

1. Убедитесь, что файл `api/sync-availability.ts` существует
2. Проверьте, что он в репозитории (git ls-files api/)
3. Проверьте логи деплоя в Vercel
4. Используйте правильный URL проекта
